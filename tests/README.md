# System Tests

End-to-end tests for Use Case 3. PDF files are **not committed** to the
repository; download them from arXiv before running PDF-dependent tests.
Committed reference outputs under `tests/papers/artifacts/` support offline
review without PDFs.

## Test PDF fixtures

| Paper | Local path | Source |
| --- | --- | --- |
| DrQ-v2 | `tests/papers/drq_v2/2107.09645v1.pdf` | https://arxiv.org/pdf/2107.09645v1 |
| SIGA | `tests/papers/siga/SIGA_Self_Evolving_Coding_Agent_Adapters_for_Scientific_Simulation.pdf` | https://arxiv.org/pdf/2606.09774v1 |
| Transformer | `tests/papers/transformer/1706.03762v7.pdf` | https://arxiv.org/pdf/1706.03762v7 |
| BERT | `tests/papers/bert/1810.04805v2.pdf` | https://arxiv.org/pdf/1810.04805v2 |
| RAG | `tests/papers/rag/2005.11401v4.pdf` | https://arxiv.org/pdf/2005.11401v4 |

Example:

```bash
mkdir -p tests/papers/drq_v2
curl -L -o tests/papers/drq_v2/2107.09645v1.pdf \
  https://arxiv.org/pdf/2107.09645v1
```

## Prerequisites

- Python dependencies from root `requirements.txt`
- Validated PDF-NLP runtime models under `modules/pdf_nlp/models/runtime/`
  (via `setup_assets.py`)
- `modules/dataset/data/processed/dev_5k.jsonl` (and `dev_sample.jsonl` for fast retrieval)
- **Ollama** with `qwen3:8b` for summarization, peer review, and integration CLI tests

```bash
ollama pull qwen3:8b
```

## Run

```bash
./tests/run_system_tests.sh
```

Environment:

| Variable | Effect |
|----------|--------|
| `SKIP_OLLAMA=1` | Parse + retrieval E2E only |
| `REQUIRE_OLLAMA=1` | Fail if Ollama missing (no skip) |
| `COMP8420_OLLAMA_MODEL` | Override default model name |
| `COMP8420_PDF_NLP_PYTHON` | Python executable containing the PDF-NLP model dependencies |

## Layout

- `papers/` — test PDFs (download locally; see table above)
- `papers/artifacts/` — committed reference evidence (never overwritten by tests)
- `logs/artifacts/` — ignored artifacts generated by the current E2E run
- `prompts/` — topic queries and PDF task prompts
- `logs/` — per-run session JSONL + copied outputs (gitignored)
- `harness/` — paths, runners, logging, bootstrap

## Logging

Production runs write structured events under
`integration/data/sessions/<run-id>/`; the harness can mirror those rows into
`tests/logs/`. See [`docs/OBSERVABILITY.md`](../docs/OBSERVABILITY.md).

The suite covers parser-only operation, full PDF-NLP enrichment, title-derived
retrieval, integration CLI paths, async API jobs, frontend build behavior, and
temporary-upload deletion. Ollama paths are skipped only when explicitly requested.

## Scope note

Integration tests use `--corpus-limit 200` or `dev_sample.jsonl`. This is a
**test scope**; it does not support broad production superiority claims.
